Metode K-Nearest Neighbors (KNN) dalam deteksi outlier#

K-Nearest Neighbors (KNN) adalah algoritma berbasis jarak yang sering digunakan dalam klasifikasi, regresi, dan deteksi outlier. Dalam deteksi outlier, KNN mengukur kedekatan suatu titik data dengan tetangga terdekatnya. Jika suatu titik data memiliki jarak yang jauh dari sebagian besar titik lain, maka dapat dianggap sebagai outlier.

Metode K-Nearest Neighbors (KNN) dalam deteksi outlier bekerja dengan mengukur jarak antara setiap titik data dengan k tetangga terdekatnya, titik data yang memiliki jarak rata-rata yang signifikan lebih besar dibandingkan dengan mayoritas titik data lainnya diidentifikasi sebagai outlier. Pemilihan nilai k sangat krusial karena memengaruhi sensitivitas metode ini, di mana nilai k yang terlalu kecil dapat melewatkan outlier dan nilai yang terlalu besar dapat salah mengklasifikasikan kluster kecil sebagai outlier.

metode untuk mengukur jarak lurus (garis terpendek) antara dua titik dalam ruang Euclidean. Jarak Euclidean diberikan oleh rumus:

\(d = \sqrt{(x_2 - x_1)^2 + (y_2 - y_1)^2}\)

Konsep ini dapat diperluas ke dimensi yang lebih tinggi dengan menjumlahkan selisih kuadrat setiap koordinat sebelum diakarkan. Euclidean Distance sering digunakan dalam algoritma clustering, k-nearest neighbors (KNN) dll

# Fungsi untuk mendeteksi dan memfilter outlier menggunakan LOF
def detect_and_filter_outliers(df, n_neighbors=20, contamination=0.05):
    """
    Mendeteksi outlier menggunakan Local Outlier Factor (LOF).
    Args:
        df: DataFrame yang berisi data numerik.
        n_neighbors: Jumlah tetangga yang dipertimbangkan dalam LOF.
        contamination: Persentase data yang diperkirakan sebagai outlier.

    Returns:
        DataFrame yang berisi data tanpa outlier.
    """
    numeric_df = df.select_dtypes(include=[np.number])  # Pilih hanya kolom numerik

    if numeric_df.empty:
        print("DataFrame tidak memiliki kolom numerik untuk analisis outlier.")
        return df

    lof = LocalOutlierFactor(n_neighbors=n_neighbors, contamination=contamination)
    outlier_labels = lof.fit_predict(numeric_df)

    df["outlier"] = outlier_labels  # -1 untuk outlier, 1 untuk data normal

    outliers = df[df["outlier"] == -1]
    filtered_df = df[df["outlier"] == 1].drop(columns=["outlier"])

    print("Data Outlier:")
    print(outliers)

    print("\nData yang sudah difilter:")
    print(filtered_df)

    return filtered_df

# Fungsi untuk menggabungkan data dari PostgreSQL dan MySQL
def merge_filtered_data(df_pg, df_mysql):
    """
    Menggabungkan dua DataFrame dari PostgreSQL dan MySQL setelah filtering outlier.

    Args:
        df_pg (DataFrame): Data dari PostgreSQL yang sudah difilter.
        df_mysql (DataFrame): Data dari MySQL yang sudah difilter.

    Returns:
        DataFrame yang sudah digabungkan.
    """
    # Pastikan kolom yang sama sebelum menggabungkan
    common_columns = list(set(df_pg.columns).intersection(set(df_mysql.columns)))

    df_pg = df_pg[common_columns]
    df_mysql = df_mysql[common_columns]

    # Menggabungkan data
    merged_df = pd.concat([df_pg, df_mysql], ignore_index=True)

    print("\nData Gabungan Setelah Filtering:")
    print(merged_df)

    return merged_df

# 🔹 Contoh Penggunaan
df_pg = get_pg_data()

print("\n🔎 Analisis Outlier untuk Data MySQL:")
filtered_mysql = detect_and_filter_outliers(df_mysql)

# 🔹 Menggabungkan data yang sudah difilter
merged_data = merge_filtered_data(filtered_pg, filtered_mysql)

# 🔹 Simpan hasil ke file CSV (opsional)
merged_data.to_csv("merged_filtered_data.csv", index=False)
